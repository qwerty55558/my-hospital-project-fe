> 본 문서는 안과 전문 병원 홍보용 반응형 웹사이트 신규 구축을 위한 1차 기획안입니다. 범위, 일정, 산출물, 정보구조, CMS 설계, 디자인·개발 가이드를 포함합니다.
> 

---

### 1) 프로젝트 개요

- 목적: 병원의 핵심 정보(의료진, 진료 분야, 오시는 길, 진료시간, 병원 소식)를 명확하고 신뢰감 있게 제공
- 범위: 기획 · UI/UX 디자인 · 프런트엔드 · 백엔드(CMS) · 배포 · 기본 SEO · 접근성 기본 준수
- 제외: 실시간 예약 시스템(본 범위 외)
- 지원 디바이스: PC · 모바일 반응형
- 벤치마킹: 김포우리안과, 이미지안과의원, 함소아한의원

### 2) 핵심 성공 지표(KPI)

- 사용자: 핵심 정보 3클릭 이내 도달률 90% 이상
- 성능: LCP 2.5s 이하, CLS 0.1 이하, 페이지 무게 1.5MB 이하(홈)
- 검색: 브랜드 키워드 상위 노출 및 기본 구조화 데이터 적용
- 운영: CMS로 전 콘텐츠(의료진/진료/소식 등) 자가 수정 가능

### 3) 정보 구조(IA) · 사이트맵

- 홈
    - 핵심 강점 하이라이트, 주요 진료 과목 바로가기, 공지/소식 카드, 빠른 연락/오시는 길
- 병원 소개
    - 인사말, 미션·비전, 연혁, 시설/장비 스냅샷
- 의료진 소개
    - 의료진 리스트 → 상세(프로필, 전문 분야, 경력, 학회, 진료 스케줄)
- 진료 안내
    - 질환/치료 카테고리: 안과 질환, 시력교정, 특수 클리닉 등 → 각 상세 페이지(증상, 진단, 치료, 자주 묻는 질문)
- 이용 안내
    - 진료시간, 오시는 길(카카오맵/네이버지도), 주차/접수 안내, 유의사항
- 병원 소식
    - 공지사항, 언론보도, 이벤트
- 온라인 상담
    - 문의 등록 폼, 목록, 상세, 비공개 처리 옵션(운영정책에 따라 표시 범위 설정)
- 푸터(공통)
    - 주소, 대표전화, 사업자 정보, 이용약관, 개인정보처리방침, SNS 링크

### 4) 주요 화면 와이어프레임(텍스트 가이드)

- 홈: 히어로 섹션(한 줄 가치제안 + 대표 이미지) → 주요 진료 3~6개 카드 → 의료진/장비 신뢰 섹션 → 공지 카드 3개 → 오시는 길 요약 + 지도 모달 버튼 → 하단 콜투액션(문의, 전화)
- 의료진 리스트: 썸네일, 이x름, 전문분야 태그, 간단 경력 → 필터(전문 분야)
- 의료진 상세: 프로필, 학력/경력, 진료분야, 학회 활동, 진료 스케줄, 관련 진료 링크
- 진료 상세: 증상 개요 → 진단 방법 → 치료 옵션 → 수술/시술 안내(있다면) → 주의사항 → FAQ
- 이용 안내: 진료시간 테이블, 지도 임베드, 대중교통/주차, 연락처
- 소식: 카드 목록, 태그/검색, 상세 본문(이미지 포함)

### 5) CMS 설계(초안)

- 컬렉션: 의료진(doctors)
    - 이름, 직함, 전문 분야(태그), 프로필 이미지, 경력 요약, 학력/학회, 진료 스케줄
- 컬렉션: 진료 안내(services)
    - 카테고리, 제목, 요약, 본문(증상/진단/치료/FAQ 섹션), 대표 이미지, 관련 의료진(참조)
- 컬렉션: 병원 소식(posts)
    - 유형(공지/보도/이벤트), 제목, 본문, 썸네일, 게시일, 노출여부
- 컬렉션: 장비(equipments)
    - 이름, 용도, 이미지, 설명, 관련 진료(참조)
- 공통: 메타 정보(SEO 제목/설명, 오픈그래프 이미지), 슬러그, 태그, 정렬/노출 옵션

### 6) 디자인 가이드(톤앤매너)

- 키워드: 신뢰, 안정, 선명함, 편안함
- 컬러: 메인 네이비/블루 + 화이트 공간 비중 확대, 포인트 컬러 절제 사용
- 타이포: 가독성 중심의 산세리프(국문 가독 최적화), 본문 16px 이상
- 컴포넌트: 카드형 리스트, 아이콘 기반 안내, 명확한 버튼 상태, 충분한 대비율(AA 이상)
- 접근성: 시맨틱 마크업, 키보드 포커스, 대체 텍스트, 폼 레이블, 색상 대비 기준 충족

### 7) 기술 스택 제안(예시)

- 프런트엔드: Next.js 14(App Router), TypeScript, Tailwind CSS, SWR
- CMS: Headless(예: Strapi/Directus) 또는 Next.js CMS 레이어(선호도·운영여건에 따라 결정)
- 배포: Vercel 또는 AWS(CloudFront + S3 + Lambda) 선택지 제시
- 지도: 카카오맵 SDK, 네이버지도 대안 제공
- 분석/로그: Google Analytics v4, 서버 로그(CloudWatch 등)
- SEO: 메타 태그, sitemap.xml, robots.txt, 구조화 데이터(Organization, MedicalClinic, Physician)

### 8) 개발 범위 상세 및 기능 명세

- 공통
    - 반응형, 다국어 대비 구조 가능(초기 한국어 단일), 성능 최적화, 웹 접근성 준수
- 사용자 웹사이트
    - 검색(선택), 카테고리/태그 필터, 관련 콘텐츠 위젯, FAQ 토글
- 온라인 상담
    - 문의 등록 폼, 관리자 승인 후 노출 옵션, 비공개 처리
- 관리자(CMS)
    - 의료진/진료/장비/소식 CRUD, 미디어 업로드, 미리보기, 권한(관리자/에디터)


### 11) 운영·유지보수(초안)

- 경미 수정 대응, 보안/라이브러리 업데이트, 장애 대응 절차, 백업 정책

### 12) 리스크 · 대응

- 콘텐츠 지연 → 수집 템플릿 제공, 마감 리마인드
- 의료 자문 필요 → 내부 검토 라우팅, 임시 노출 정책
- 사진·이미지 저작권 → 촬영/라이선스 명시

### 13) 필요 자료 체크리스트

- 병원 소개 텍스트, 로고/BI, 컬러 규정, 의료진 정보(프로필·경력·사진), 진료 항목별 콘텐츠, 위치/주차 안내, 기존 보도자료/공지
